#### API: KUDAB.RU — СПЕЦИФИКАЦИЯ, ПРИНЦИПЫ, СЦЕНАРИИ

---

### ОБЩЕЕ ОПИСАНИЕ

**kudab.ru** предоставляет публичный и партнерский REST API (в перспективе — GraphQL), который обеспечивает доступ к основной базе данных событий, интересов, сообществ, пользователей, а также к сценариям взаимодействия с ботами, рассылками и внешними сервисами.  
API ориентирован как на массовых пользователей (B2C, приложения, PWA), так и на интеграцию с внешними B2B-сервисами (городские порталы, биллинги, тур- и образовательные платформы).

- **Single Source of Truth:** Все сервисы платформы используют единый API — нет расхождений между фронтендом, ботом, мобильной и внешними партнерами.
- **Безопасность:** JWT Auth, OAuth2 (B2B), CORS, rate limiting, логирование.
- **Версионирование:** `/api/v1/*` — минимизация breaking changes.

---

### СТРУКТУРА ЭНДПОИНТОВ

**Основные группы:**

- `/api/v1/users` — профили пользователей, регистрация, интересы, авторизация через Telegram/email.
- `/api/v1/telegram` — авторизация, получение/привязка telegram_id, взаимодействие с ботом.
- `/api/v1/events` — события, афиша, фильтры, календарь, RSVP (“пойду”/“избранное”), интеграция с календарями.
- `/api/v1/communities` — сообщества, их афиши, участники, интересы.
- `/api/v1/interests` — категории, теги, персональные рекомендации.
- `/api/v1/posts` — контент, посты, вложения, комментирование.
- `/api/v1/attachments` — получение/загрузка вложений.
- `/api/v1/notifications` — push- и telegram/email-уведомления.
- `/api/v1/integrations` — парсеры, внешние сервисы, публикация событий через API.

### ПРИМЕРЫ ЭНДПОИНТОВ

#### Пользователь и Telegram

- `POST /api/v1/users/register`
- `POST /api/v1/users/login`
- `POST /api/v1/telegram/auth`
- `GET /api/v1/users/{id}`
- `GET /api/v1/telegram/me` — информация по Telegram-пользователю
- `POST /api/v1/telegram/bind` — привязка Telegram к основному аккаунту

#### События и афиша

- `GET /api/v1/events` — поиск и фильтрация (по дате, интересу, гео, ключевому слову, статусу)
- `GET /api/v1/events/calendar` — календарь событий (структура для интеграции с календарями, PWA)
- `GET /api/v1/events/{id}` — подробная информация
- `POST /api/v1/events/{id}/rsvp` — “пойду”/“избранное” (нет лайков!)
- `GET /api/v1/events/favorites` — избранные события пользователя
- `POST /api/v1/events/propose` — добавление своего события (авто/ручная модерация)

#### Сообщества

- `GET /api/v1/communities`
- `GET /api/v1/communities/{id}`
- `GET /api/v1/communities/{id}/events`
- `GET /api/v1/communities/{id}/members`

#### Интересы и рекомендации

- `GET /api/v1/interests`
- `POST /api/v1/interests/select` — выбор интересов пользователем
- `GET /api/v1/interests/recommend` — рекомендации событий и сообществ

#### Контент и вложения

- `GET /api/v1/posts`
- `GET /api/v1/posts/{id}`
- `POST /api/v1/posts/{id}/comment`
- `GET /api/v1/attachments/{id}`

#### Интеграция и внешние сервисы

- `POST /api/v1/integrations/event` — публикация события из внешнего источника
- `GET /api/v1/integrations/events` — выгрузка афиши для партнеров
- `GET /api/v1/integrations/export.ics` — экспорт событий в календарь (iCal)

### ПРИНЦИПЫ И РЕКОМЕНДАЦИИ

- **Стандарты:** JSON API (RFC 8259), OpenAPI 3.0, корректные HTTP-статусы и коды ошибок.
- **Пагинация, фильтрация, сортировка** на всех списочных эндпоинтах.
- **Ограничение доступа**: ключевые операции — только для авторизованных; модерация — по ролям.
- **Webhooks**: для интеграций с внешними сервисами и рассылками.
- **Push/PWA**: ответы оптимизированы для мобильных интерфейсов и офлайн-режима (если PWA).

### ОСНОВНЫЕ СЦЕНАРИИ

1. **Пользователь через Telegram-бота:**
    - Впервые заходит в бота — создается или находит профиль в базе.
    - Выбирает интересы — бот отправляет их через API, формируется персональная афиша.
    - “Пойду” — RSVP на событие через API, добавляется в избранное и синхронизируется с календарем.
    - Получает уведомления о новых событиях в своих интересах.

2. **Внешний партнер (городской портал, туристическая платформа):**
    - Получает свежую афишу через `GET /api/v1/integrations/events`.
    - Публикует события из собственной системы через `POST /api/v1/integrations/event`.

3. **Организатор:**
    - Добавляет свое событие через API или web-кабинет.
    - Следит за откликами (RSVP), получает аналитику по посещениям.

### КОНКУРЕНТНЫЕ ПРЕИМУЩЕСТВА API

- Автоматическая и ручная агрегация событий (в т.ч. парсинг соцсетей)
- Открытость для партнеров (легкая интеграция)
- Поддержка календарей (Google, iCal)
- Универсальность — единый формат данных для всех типов событий и контента
- Моментальное обновление афиши (real-time через webhooks или polling)
